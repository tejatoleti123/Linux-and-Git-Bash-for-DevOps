# AWS EC2 Setup and SSH Access (Free Tier)

This guide explains how to create a free AWS account, launch an Amazon EC2 instance using Amazon Linux, and connect to it securely using SSH via MobaXterm.


## Step 1: Create a Free AWS Account

1. Go to https://aws.amazon.com
2. Click **Create an AWS Account**
3. Sign up using an email address
4. Add billing information (required for verification)
5. Choose the **Free Tier** plan
6. Complete identity verification

AWS provides a 12-month free tier that includes limited EC2 usage.


## Step 2: Log in to the AWS Management Console

1. Sign in to the AWS Console
2. Select a region (example: **us-east-1**)
3. Navigate to **EC2** from the Services menu


## Step 3: Launch an EC2 Instance (Amazon Linux)

1. Click **Launch Instance**
2. Name the instance (example: `devops-linux-ec2`)
3. Choose **Amazon Linux** as the AMI
4. Select instance type:
   - `t2.micro` (Free Tier eligible)
5. Create a new key pair:
   - Key type: RSA
   - Download the `.pem` file securely
6. Configure network settings:
   - Allow **SSH (port 22)**
7. Click **Launch Instance**


## Step 4: Get EC2 Public IP Address

1. Go to **EC2 → Instances**
2. Select your instance
3. Copy the **Public IPv4 address**

This IP is used for SSH access.



## Step 5: Download and Install MobaXterm

1. Open a browser and go to:
   https://mobaxterm.mobatek.net

2. Click **Download**
3. Choose **Home Edition (Free)**
4. Download the **Installer version**
5. Run the installer and complete the setup using default options

MobaXterm provides an SSH client and Unix-like tools on Windows, making it suitable for connecting to Linux EC2 instances.



## Step 6: Connect to EC2 Using MobaXterm

1. Open **MobaXterm**
2. Click **Session → SSH**
3. Enter:
   - Remote host: EC2 public IP
   - Username: `ec2-user`
4. Enable **Use private key**
5. Select the downloaded `.pem` file
6. Click **OK** to connect

If successful, you will see a terminal connected to the EC2 instance.


## Step 7: Verify Connection on the EC2 Instance

Run the following commands after login:

whoami
